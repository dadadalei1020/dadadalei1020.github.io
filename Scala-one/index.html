public class MyList {
    private List<String> lst;

    public MyList(){}
    public MyList(List<String> lst) {
        this.lst = lst;
    }

    /**
     *  参数是一个接口
     *  定义map方法的时候，参数是一个接口
     * @param mp
     * @return
     */
    public List<String> map(MapFunction mp){
        ArrayList<String> tmp = new ArrayList<>();
        for(String str:lst){
            // 执行某一项业务逻辑，返回一个新的字符串
            String newStr = mp.transform(str);
            tmp.add(newStr);
        }
        return tmp;
    }
}

9.2. 函数

函数类似于方法。同样可以是一段业务逻辑的综合。

函数有2种定义方式：简写;复杂的写法

函数的关键字 =>

9.2.1. 第一种定义方式：

val /var 函数的名称 = (函数的参数列表) => {函数体}

val f1 = (a:Int,b:Int) => a + b

这种定义方式，函数的返回值类型，会自动推导。根据函数体的返回值（最后一行的值）

scala> val f1 = (a:Int,b:Int) => a + b f1: (Int, Int) => Int = scala> val f1 = (a:Int,b:Int) => a > b f1: (Int, Int) => Boolean =

定义函数之后，返回的是函数的签名信息：

函数的签名： 函数的名称 输入参数的类型及个数 函数的返回值类型

函数签名实际上就是定义好了函数的规范。

Function实际上就是java中的接口。

匿名函数：

scala> (a:Int,b:Int) => if(a>b)a else b res20: (Int, Int) => Int =

9.2.2. 第二种定义方式：

scala> val f1 = (a:Int,b:Int) => a + b

f1: (Int, Int) => Int =

定义方式：

val /var 函数名称:(输入参数类型) => 返回值类型 = (输入参数的引用) => 函数体

val f1:(Int,Int) => Int = (a:Int,b:Int) => a + b

简写为：

val f1:(Int,Int) => Int = (a,b) => a + b

val f1 = (a:Int,b:Int) => a + b def main(args: Array[String]): Unit = { // 函数的调用形式，同方法的调用 val res: Int = f1(10,20) println(res) }
